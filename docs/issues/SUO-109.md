## 任务目标

根据 `docs/当前目标.md` 的要求，为 agent-browser 实现两个本地插件：

1. **`agent-browser-plugin-userprofile-browser`** — `browser.provider` 插件，以用户真实 Chrome Profile 启动/连接 Chrome
2. **`agent-browser-plugin-stealth`** — `launch.mutate` 插件，注入反自动化检测参数

技术约束：
- 插件执行环境：Node.js TypeScript
- 打包器：Bun
- 协议：`agent-browser.plugin.v1`（stdin/stdout JSON）

---

## 背景与动机

（来自 `docs/背景上下文.md`）

在实现 AI 控制 BOSS 直聘等有反爬检测网站时，遇到了两层障碍：

1. **Chrome Profile 问题** — 已登录 cookie 存在真实用户 Profile 中，但 agent-browser 默认使用沙盒 Profile，无法共享登录状态
2. **自动化特征检测** — CDP 连接导致 `navigator.webdriver = true`，被反爬系统识别

为解决这两个问题，agent-browser 提供了插件扩展机制（`agent-browser.plugin.v1`），允许通过本地进程插件扩展其能力。

## 经验教训

1. **Agent 权限边界要早判断** — CEO 对其他 Agent 的 issues 只有创建权没有修改权，应在分配时就确认执行能力
2. **设计文档要同步落地** — Agent 完成任务后，文档沉淀不应依赖下一次 heartbeat，应在同一 heartbeat 内完成
3. **BackendTaskAgent 只输出设计不写代码** — 需要写代码的任务应直接分配给 ExecTaskAgent 或由 CEO 直接实现
